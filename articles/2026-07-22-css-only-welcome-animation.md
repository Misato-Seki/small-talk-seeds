# framer-motionなしでウェルカムメッセージのふわっとアニメーションを作った過程

トップページを開いたときに「一緒に働く仲間との雑談のタネを見つけて、会話に花を咲かせましょう」というウェルカムメッセージを、🌱→🌸のイラスト切り替えつきで、ふわっと表示してふわっと消す、というアニメーションを作った。ライブラリ(framer-motionなど)は使わずCSSだけで実現する方針にしたところ、いくつかの箇所で詰まったので、過程をまとめる。

## やりたかったこと

- メッセージ全体が、ふわっと表示されて、4秒後にふわっと消える
- 表示中の中身は2段階に切り替わる(🌱タネのメッセージ → 🌸花のメッセージ)
- 依存パッケージを増やさず、CSSの`@keyframes`だけで完結させる

## 設計方針: 秒数を`%`に変換したタイムラインを作る

`@keyframes`は`0%`〜`100%`でしか時間を表現できないので、「4秒のアニメーション」の中の「2秒経過時点」のような指定は、`animation-duration`を基準に`%`へ変換して考える必要があった。

今回は以下の3本の`@keyframes`を用意する設計にした。

- 枠全体の表示/非表示用(`0%`〜`8%`でフェードイン、`90%`〜`100%`でフェードアウト)
- 🌱タネのテキスト用(前半だけ`opacity: 1`)
- 🌸花のテキスト用(後半だけ`opacity: 1`)

3本とも同じ`animation-duration`で動かすことで、同じタイムライン上でタイミングを揃えられる。

## 詰まった箇所1: `absolute`で中央寄せしたつもりが中央に来ない

### 起きた問題

```
absolute top-0 left-0 translate-x-1/2
```

というクラスで中央寄せを試みたが、要素は左上寄りのままだった。

### 原因

`top-0 left-0`で要素の左上を親の左上に固定してしまっており、`translate-x-1/2`は「自分の幅の50%だけ右に動かす」効果しかないため、中心には来ない。

### 解決方法

中央寄せの定石である

```
top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2
```

に変更した。「まず`top-1/2 left-1/2`で要素の基準点を親の中心に置き、次に`-translate-x-1/2 -translate-y-1/2`で自分の幅・高さの半分だけ戻す」という2ステップの考え方を理解していなかったのが原因だった。

## 詰まった箇所2: `@keyframes`を書いてクラスを当てたのに何も変化しない

### 起きた問題

`app/globals.css`に自作の`@keyframes`を定義し、`animate-frame-fade-in`のようなクラス名を要素に当てたが、ブラウザで見た目が全く変化しなかった。

### 原因

2つの原因が重なっていた。

1. Tailwindの`animate-◯◯`(角括弧なし)は、Tailwindのテーマに登録された`--animate-◯◯`という変数を探しにいく仕組みで、自作の生の`@keyframes`を直接参照するには`animate-[名前_時間_タイミング関数_fill-mode]`という**任意値(角括弧)構文**を使う必要があった。
2. 途中でキーフレーム名を`frame-fade-in`から`frame-fade-in-out`にリネームしたが、コンポーネント側のクラス名を更新し忘れており、名前が一致していなかった。

### 解決方法

```
animate-[frame-fade-in-out_4s_ease-in-out_forwards]
```

のように任意値構文で書き、`globals.css`側のキーフレーム名とクラス側の参照名を一致させた。あわせて、繰り返し再生される`infinite`を指定していた箇所も、「1回だけ再生して、終わった後は最終状態(消えた状態)を保つ」設定に変更した。

## 詰まった箇所3: `animation-delay`と`animation-duration`の混同

### 起きた問題

秒数を調整しようとして`animation-delay-4000`という(存在しない)クラスを使おうとした。

### 原因

`animation-delay`は「アニメーションの開始そのものを何秒後にするか」を決めるだけで、`@keyframes`の`0%`〜`100%`を実時間の何秒に引き伸ばすかを決めるのは`animation-duration`の役割だと理解していなかった。今回はキーフレーム内の`%`で時間配分をすでに設計していたので、`delay`は不要だった。

### 解決方法

`animation-delay`は使わず、`animation-duration`(`4s`)だけを設定するようにした。

## 詰まった箇所4: 秒数を変えるたびに3箇所を書き換える手間

### 起きた問題

枠・タネ・花の3つの`animate-[...]`それぞれに秒数(`4s`)をハードコードしていたため、アニメーションの長さを変えたいたびに3箇所を書き換える必要があり、面倒だった。

### 原因

同じ情報(アニメーションの長さ)が3箇所に分散していた。

### 解決方法

CSS変数(カスタムプロパティ)`--welcome-duration`を1箇所で定義し、3つの`animate-[..._var(--welcome-duration)_..._forwards]`から共通で参照するようにした。キーフレーム自体は`%`で書かれているため、変数の値を変えるだけで、タネ→花の切り替えタイミングなどの比率を保ったまま全体の長さだけを変えられる。

## まとめ

- `@keyframes`はCSS Modulesのようなスコープの仕組みがなく、グローバルなCSS(今回は`app/globals.css`)に書く必要がある
- Tailwindで自作の`@keyframes`を使うときは、素の`animate-◯◯`ではなく`animate-[名前_時間_タイミング関数_fill-mode]`という任意値構文が必要
- `animation-delay`(開始を遅らせる)と`animation-duration`(タイムラインの長さ)は役割が違う
- 複数のアニメーションを同じタイムラインで揺動かす場合、秒数をハードコードで分散させず、CSS変数に一元化するとメンテナンスしやすい
- `position: absolute`の要素は、`top`/`left`と`translate`の組み合わせ方を正しく理解しないと中央寄せが意図通りにならない
